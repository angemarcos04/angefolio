## Overview

CSPAMS is a capstone project exploring how a school can bring concern reporting and performance monitoring into one structured system without treating sensitive information casually.

## Problem

School concerns and performance records can be spread across conversations, forms, and disconnected files. That makes follow-up harder and creates privacy risks when access boundaries are unclear.

## Approach

The system is being designed around role-based workflows, privacy-conscious reporting, and structured monitoring data. Laravel provides the application foundation, PostgreSQL supports the data model, and Filament is being considered for focused administrative workflows.

## Current status

The project is in progress. System design and backend planning are active, while implementation details and capstone requirements are still being refined. It should not be read as a finished or deployed school platform yet.

## What I am learning

- Turning sensitive real-world processes into explicit permissions and states
- Designing a relational model before UI decisions harden
- Keeping a capstone scope practical while leaving room for careful workflows
